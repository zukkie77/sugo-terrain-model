# SUGO地形モデル製作手順

スポーツランドSUGOの3D地形モデルを、標高データ＋厚紙積層＋レーザーカッターで自作するための手順書です。

---

## 全体工程

| 週 | 内容 |
|----|------|
| Week 1 | 標高データ取得・QGISで等高線生成 |
| Week 2 | SVG整形・枚数確認 |
| Week 3 | FabLab訪問・レーザーカット |
| Week 4 | 積層・接着・塗装 |

---

## Week 1: 標高データ取得 → QGIS で等高線生成

### Day 1-2: 国土地理院からデータ取得

#### 1. アカウント登録

[基盤地図情報ダウンロードサービス](https://service.gsi.go.jp/kiban/) にアクセスし、「利用者登録」から無料登録します。

> ※ 旧URL（fgd.gsi.go.jp）は2026年3月にリニューアルのため廃止。上記の新URLを使用すること。
> ※ 画面が正常に表示されない場合はブラウザのキャッシュをクリアして再アクセス。

#### 2. ダウンロード

1. ログイン後「基盤地図情報 数値標高モデル」を選択
2. **「5mメッシュ（DEM5A）」** をクリック（10mメッシュは精度が低いので注意）
3. 地図上でSUGO周辺（宮城県村田町）を選択
   - 検索窓に「スポーツランドSUGO」と入力して中心を確認
4. 該当メッシュにチェックを入れてダウンロード（ZIP形式）

> ダウンロードされるファイル形式: **XML (.xml)**

---

### Day 2-3: XML → GeoTIFF 変換

#### 1. 基盤地図情報 標高DEMデータ変換ツールをダウンロード

[https://www.ecoris.co.jp/contents/demtool.html](https://www.ecoris.co.jp/contents/demtool.html) から最新版（Ver1.7.1）をダウンロード・解凍（インストール不要）。

#### 2. 変換を実行

1. 国土地理院からダウンロードしたZIPを解凍し、`.xml` ファイルを確認
2. ツールフォルダ内の **`変換結合.vbs`** をダブルクリック
3. ダイアログでXMLが入っているフォルダを選択
4. 自動処理され以下のファイルが生成される

| ファイル名 | 内容 |
|---|---|
| `merge.tif` | 標高データ（QGISに読み込むメインファイル） |
| `merge_shade.tif` | 陰影図（地形の凹凸が視覚的にわかる） |

> ⚠️ フォルダのパスに日本語・スペースが含まれていると失敗することがある。
> 例: `C:\Users\ksuzuki\sugo_dem\` のように英数字のみのフォルダを推奨。

#### 3. DEM解像度の確認方法

QGISでmerge.tifを読み込み後、レイヤを右クリック→「プロパティ」→「情報」タブでピクセルサイズを確認。

| ピクセルサイズ | 判定 |
|---|---|
| `5.5e-05` 程度 | 5mメッシュ ✓（正しい） |
| `9.0e-05` 程度 | 10mメッシュ → 再ダウンロードが必要 |

---

### Day 3-5: QGISで等高線を生成

#### 0. QGISをインストール（未インストールの場合）

[https://qgis.org](https://qgis.org) → 「Download Now」→ Windows版（無料）をインストール。

#### 1. GeoTIFFを読み込む

```
QGISを起動
→「レイヤ」→「レイヤを追加」→「ラスタレイヤを追加」
→ merge.tif を選択 →「追加」
```

地図エリアにグレーの地形が表示されれば成功。

> 真っ黒に表示される場合: レイヤを右クリック→「プロパティ」→「シンボロジ」→「単バンド疑似カラー」に変更

#### 2. SUGO周辺に切り取る（クリップ）

```
「ラスタ」→「抽出」→「範囲でラスタをクリップ」
→ 地図上でSUGO周辺をドラッグして範囲指定
→ 出力ファイル名: sugo_clip.tif
→「実行」
```

#### 3. 等高線を生成

```
「ラスタ」→「抽出」→「等高線」
→ 入力レイヤ: sugo_clip.tif
→ 間隔: 10（メートル）
→ 出力ファイル名: sugo_contour.gpkg
→「実行」
```

地図上に等高線（黄色い線）が表示されれば成功。

#### 4. 等高線を平滑化する

生成直後の等高線はガタガタしているため、スムージングをかける。

```
「プロセシング」→「ツールボックス」（または Ctrl+Alt+T）
→ 検索窓に「平滑」と入力
→「ベクタジオメトリ」→「平滑化」をダブルクリック

設定:
  入力レイヤ  : sugo_contour
  繰り返し回数: 5
  オフセット  : 0.25
→「実行」
```

> 「平滑化」が見つからない場合は検索窓に「smooth」と英語で検索する。

#### 5. DXFでエクスポート

```
平滑化レイヤを選択した状態で
「プロジェクト」→「インポート/エクスポート」→「地図をDXFにエクスポート」
→ ファイル名: sugo_contour_final.dxf
→「OK」
```

## 既知の問題・注意事項

### 山頂・尾根が多い地形でのパーツ分離問題

→ [Issue #1](https://github.com/zukkie77/sugo-terrain-model/issues/1) 参照

SUGOの地形は稜線が複数方向に伸びており、高い層ほど孤立した小片が多数発生する。
以下の代替案を検討中：

1. **範囲を絞る**: サーキットと周辺の谷のみを対象にし、山頂部分を除外する
2. **3Dプリントに切り替える**: GeoTIFFをBlenderで3Dデータ化しFDMプリンタで出力
3. **Blenderデジタル完結**: 物理モデルを作らずレンダリング画像として完成

#### 分離した小片の位置合わせ対策（厚紙積層を続ける場合）

全層のDXFデータに共通の基準穴（直径3mm の円）を追加しておく。
積層時に竹串を通して位置合わせの基準にし、接着後に穴を埋める。
